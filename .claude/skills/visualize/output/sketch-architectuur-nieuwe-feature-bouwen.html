Deel B · in de praktijk

7 stappen: feature "Announcements" bouwen

Scenario: team-aankondigingen die op het dashboard verschijnen (titel, body, wie heeft 'm geplaatst, datum).

Fictief voorbeeld — puur om alle 7 stappen te laten zien.

1

Database migratie

Horizontaal · shared schema
supabase/migrations/20260425_announcements.sql

Tabel announcements met kolommen id · title · body · author_id · created_at. RLS policy erbij. Eén database = één schema, gedeeld door alle apps.

2

Types regenereren

Horizontaal · auto-gegenereerd
packages/database/src/types/database.ts

npm run types:generate → Supabase levert TypeScript-types voor de nieuwe tabel. Automatisch. Niet handmatig editen.

3

Queries (lezen)

Horizontaal · gedeeld
packages/database/src/queries/announcements.ts

listAnnouncements(), getAnnouncementById(). Horizontaal omdat zowel cockpit als portal straks aankondigingen kunnen tonen.

4

Mutations (schrijven)

Horizontaal · gedeeld
packages/database/src/mutations/announcements.ts

insertAnnouncement(), deleteAnnouncement(). Dezelfde reden als queries: gedeeld, dus horizontaal.

5

Feature map — het hart

Verticaal · app-specifiek
apps/cockpit/src/features/announcements/
announcements/
├─ actions/
│ └─ create.ts
├─ components/
│ ├─ announcement-card.tsx
│ └─ announcement-form.tsx
└─ index.ts ← de deur

Alle UI + actions bij elkaar. index.ts exporteert alléén wat de route nodig heeft:

export { AnnouncementCard, AnnouncementForm } from './components/...'
export { createAnnouncement } from './actions/create'
6

Route (URL)

Next.js dicteert
apps/cockpit/src/app/(dashboard)/announcements/page.tsx

Ultra-dun bestand — alleen composeren, niet uitvoeren:

import {'{'} AnnouncementCard, AnnouncementForm {'}'} from '@/features/announcements'
import {'{'} listAnnouncements {'}'} from '@repo/database/queries/announcements'
// Server Component haalt data op, rendert de feature-componenten
7

Dashboard-integratie

Verticaal · via de deur
apps/cockpit/src/features/dashboard/components/announcements-card.tsx

De dashboard-feature hergebruikt de announcements-componenten via de index.ts — niet via directe paden:

// ✓ Via de deur
import {'{'} AnnouncementCard {'}'} from '@/features/announcements'
// ✗ Niet zo — dat is door de muur stappen
import {'{'} AnnouncementCard {'}'} from '@/features/announcements/components/announcement-card'

🎯 Wat je nu hebt

  • ✓ Eén pagina om op te pakken: /announcements
  • ✓ Eén map om de feature te begrijpen: features/announcements/
  • ✓ Eén import voor hergebruik: '@/features/announcements'
  • ✓ Portal kan meekijken zonder cockpit-code te kennen — via @repo/database/queries/announcements

Deel C · spiekbriefje

"Waar komt dit bestand?"

Vier vragen — volg de eerste die met "ja" wordt beantwoord.

1
Is het een database-wijziging?
Migratie, RLS policy, seed data
→
supabase/migrations/
2
Wordt het door meerdere apps gebruikt?
Queries, mutations, agents, shared UI
→
packages/ · horizontaal
3
Is het een URL-route?
Pagina, layout, loading, error state
→
apps/[app]/src/app/ · Next.js
4
Anders — alles wat bij één feature hoort
Server Action, feature-component, feature-specifieke helper
→
apps/[app]/src/features/[naam]/ · verticaal

De kern

Geen beslissingsmoeheid meer.

Als je de vier vragen doorloopt, weet je precies waar elk bestand heen gaat. AI weet het ook. De codebase vertelt zichzelf.
