Beslispunt — vóór we beginnen

index.ts per submap — ja of nee?

Optie A — Met index.ts Advies
import {'{ ... }'} from '@/actions/themes'
  • ✓ Duidelijke "deur" per feature
  • ✓ Dwingt publieke API af
  • ✓ Ousterhout "deep modules"
  • △ Iets zwaardere bundel (Next.js 16 mitigeert)
Optie B — Zonder index.ts Alternatief
import {'{ ... }'} from '@/actions/themes/crud'
  • ✓ Snellere dev-builds
  • ✓ Bulletproof React-stijl
  • △ Elke file is impliciet "publiek"
  • △ Consumer moet exact bestand weten
Claude's advies voor jullie context: Optie A. Jij bent non-coder + AI-gedreven — "één duidelijke ingang per feature" is meer waard dan een paar ms build-snelheid.
